# Architecture Decision Framework

CTO-level framework for making architectural decisions with trade-off analysis.

## Purpose

Helps choose architecture patterns based on actual needs (not trends). Includes decision tree, ADR templates, and anti-pattern warnings.

## For Phúc SA

Use when:
- Choosing between monolith vs microservices
- Deciding on DDD vs Transaction Script
- Evaluating Repository pattern necessity
- Documenting architecture decisions (ADRs)

## Key Tools

1. **Decision Tree** - Quick pattern selection based on constraints
2. **3 Questions** - Validation before adding complexity
3. **ADR Template** - Document rationale for decisions
4. **Anti-Pattern Table** - Avoid premature optimization

## Core Principle

Start simple → Add complexity ONLY when proven necessary
