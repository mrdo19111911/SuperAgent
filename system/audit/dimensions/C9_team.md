# C9: Team & Org Capability

**Focus:** Can team sustain and evolve the codebase?

## Checklist

- [ ] **Bus Factor:** >1 person understands each critical module?
- [ ] **Knowledge Transfer:** Onboarding docs exist?
- [ ] **Code Review:** PR review process enforced?
- [ ] **Documentation:** README, architecture diagrams, setup guides?

## Red Flags

- Bus factor = 1 (single person dependency)
- No onboarding docs (new devs lost)
- No code review (quality issues)
- Legacy code no one understands

## Examples

**FRAGILE:** Only 1 dev knows payment module, no docs, no reviews
**RESILIENT:** 3+ devs per module, comprehensive docs, mandatory PR reviews
